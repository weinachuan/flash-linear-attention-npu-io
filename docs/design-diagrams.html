7. 数据库字段字典

字段名保持英文，与 D1 表结构一致；中文列说明字段的业务语义，便于研发和管理者共同评审。

Data Dictionary
tasks 任务主表
id任务唯一标识。
title事项名称，通常包含算子名和特性名。
owner责任人，可包含多个人名。
status任务状态：todo、doing、blocked、delayed、done。
risk风险等级：高、中、低，由 PR、报告、DDL、待排人力规则联动。
priority优先级：P0、P1、P2。
group_id所属转测批次或时间节点。
special_id所属专项，可为空。
start_date任务计划开始日期。
end_date任务计划完成日期，作为 DDL 判定基础。
pr_link关联 PR 链接，允许一个特性关联多个 PR。
test_report转测报告或交付件链接。
task_segments 甘特分段表
id分段唯一标识。
task_id关联任务 ID。
start_date该分段计划开始日期。
end_date该分段计划结束日期。
reason任务被中断、切分或计划调整的原因。
position同一任务内分段排序。
groups / specials 分组与专项
groups.id转测批次或里程碑 ID。
groups.title转测批次名称，例如 06-25 转测。
groups.due_date该批次对外承诺交付日期。
specials.id专项 ID。
specials.title专项名称，例如文档更新或调用示例。
specials.collapsed页面上该专项是否默认折叠。
people / users 人员与登录账号
people.name项目人员姓名，用于任务责任人、筛选、人力视图。
people.placeholder是否为系统占位人员，例如待排人力。
users.username登录账号名，初始化时默认等于人员姓名。
users.owner_name开发账号绑定的人员姓名，用于权限判定。
users.role账号角色：admin 或 developer。
users.password_hash加盐密码哈希，不保存明文密码。
audit_entries 审计日志
ts操作发生时间。
action动作类型，例如 task.patch、person.create。
entity被操作对象类型。
entity_id被操作对象 ID。
summary给页面展示和人工检索用的中文摘要。
detail结构化变更明细，JSON 格式。
project_meta 项目元数据
project项目名称、仓库、负责人等基础配置。
repoScan仓库扫描结果快照。
prCatalogPR 候选池，用于页面选择和风险联动。
stateVersion后端数据版本号，前端每 3 秒轮询用于实时刷新。
