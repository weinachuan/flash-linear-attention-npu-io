1. 快速开始

开发同学：登录后主要维护自己任务的 PR 链接 和 转测报告，同算子关联任务可查看但不可编辑。
管理员：可维护全量任务、人员、分组、专项、排期、风险、优先级和账号权限。
  1. 打开项目看板：flash-linear-attention-npu 项目看板。
  2. 只查看数据时无需登录；页面会默认展示当前时间窗口内的任务计划、风险和任务明细。
  3. 需要编辑时，在顶部工具栏输入账号和密码，点击 登录。
  4. 修改字段后，单条任务可点行内保存；多条任务可统一点 保存全部。
  5. 保存后数据写入 Cloudflare D1，其他人页面会在几秒内自动刷新或提示有新数据。

2. 三类视图

甘特计划

按任务展示计划起止时间。管理员编辑时可拖动任务条调整排期，也可在条边缘拉伸缩短或延长。

人力视图

按人员展示每天正在做什么。同一天多任务会叠放显示；中国法定节假日、周末和调休日会在日期列中标注。

算子视图

按算子聚合特性任务，一个算子一行，便于看该算子有哪些交付点、哪些特性延期或缺交付件。

时间窗口怎么用

  • 开始 和 结束 定义整个时间轴范围。
  • 绿色窗口条表示当前显示范围，初始化会铺满当前时间轴，可在轴内拖动或拉伸。
  • 前扩 7 天 和 后扩 7 天 用于扩大可观察范围。
  • 显示全量 会把时间轴重置为覆盖全部任务计划的范围。
  • 任务条末尾展示计划完成日期；延期阶段会用白底红框 delay 风格突出。

3. 任务明细

任务明细是最主要的编辑区域，支持筛选、排序、保存、PR/报告录入和任务管理。

功能说明
筛选支持按风险、优先级、事项关键字、责任人、分组、专项、状态筛选。责任人筛选支持多选。
排序点击表头可排序。默认排序是风险从高到低；风险相同时按优先级 P0 到 P2；仍相同时 DDL 更早的排在前面。
PR 链接可从 PR 候选池选择，也支持输入 PR 号快速匹配。一个 PR 可关联多个任务，一个任务也可关联多个 PR。
转测报告填写转测报告链接后，若 PR 已合入且报告齐备，状态会进入 done。
保存字段级保存，只提交变更字段，降低多人同时编辑时互相覆盖的风险。
最近变更页面底部展示最近 10 条变更，可展开查看具体字段差异。
如果保存时提示“后台已有新数据”，说明有人在你打开页面后提交过修改。此时应先刷新或手动合并，系统不会让你直接覆盖别人的修改。

4. 开发同学权限

开发账号用于让每位同学维护自己的交付件，同时避免误改项目排期和风险判断。

权限项开发同学是否可操作说明
查看任务可以进入编辑后，仅看到自己负责的任务，以及当前时间窗口内与自己任务属于同一算子的关联任务。
编辑自己任务的 PR 链接可以用于提交代码合入依据。可选 PR 候选池，也可输入 PR 号匹配。
编辑自己任务的转测报告可以用于闭环交付。报告齐备后状态可自动进入完成判断。
查看同算子关联任务可以，只读用于理解上下游和同算子交付情况，不允许编辑。
新增、删除任务不可以避免个人误删或新增无排期闭环的任务。
拖动、拉伸、切分甘特条不可以开发同学不能自行改排期。
修改风险、优先级、状态不可以这些字段由规则和管理员维护。
修改人员、分组、专项不可以组织结构由管理员维护。
修改自己的密码可以登录后点击 修改密码，输入旧密码、新密码并确认后生效。

开发同学推荐操作流程

  1. 登录后确认顶部状态显示自己的账号。
  2. 在任务明细中筛选自己的名字，或直接查看默认权限范围内的任务。
  3. 为自己负责的任务填写或选择 PR 链接。
  4. 转测完成后填写转测报告链接。
  5. 点击单行保存或保存全部。
  6. 如果发现任务责任人、排期、风险、优先级不正确，联系管理员调整。

5. 管理员权限

管理员账号用于维护项目主数据和权限数据，应避免多人共用同一个管理员账号长期操作。

模块管理员能力关键规则
任务新增、编辑、删除、保存全部、调整 PR/报告、风险、优先级、状态、责任人等字段。删除任务属于高风险操作，会弹窗确认。
排期拖动任务条、拉伸任务条、切分任务、修改起止日期。改任务排期必须填写变更原因，否则不允许保存。
人员新增、编辑、删除真实人员。待排人力 是系统占位，不在人员管理中编辑。删除人员前必须先解除其任务关联。
分组新增、编辑、删除转测分组。删除分组不会删除任务；关联任务和专项会转入默认分组。
专项新增、编辑、删除专项。删除专项不会删除任务；关联任务的专项字段会变为普通事项。
账号创建开发账号、创建管理员账号、升级角色、重置密码。重置密码属于高风险动作，应通过受控 Actions/script 执行，并单独转发密码。

高风险操作确认

  • 删除任务、删除人员、删除专项、删除分组、修改密码都会弹出确认。
  • 改排期必须备注原因，原因会写入任务备注或甘特分段原因。
  • 后台版本变化时保存会被拒绝，需要刷新或人工合并。

6. 状态、风险、优先级规则

状态

  • delay：超过 DDL，且交付件未完备。
  • done：交付件完备。
  • doing：未超过 DDL，且已规定起始闭环时间、正在推进。
  • todo：待排人力，或只有事项但未规定闭环时间。
  • Pending：更高优先级事项插入，导致任务因人力问题挂起。

风险

  • 低：PR 已合入；或有未合入 PR 且距离 DDL 大于 5 天。
  • 中：有未合入 PR 且距离 DDL 小于等于 5 天；或无 PR 但距离 DDL 大于 10 天。
  • 高：无 PR 且距离 DDL 小于等于 10 天；或责任人为待排人力/待填写。

优先级

  • P0：有对外承诺的事项。
  • P1：无对外承诺但收益较高、涉及易用性，或内部目标早于对外交付承诺。
  • P2：不阻塞使用、收益较小、自规划类型事项。
风险和状态会根据 PR、转测报告、DDL、责任人等信息联动更新。管理员仍可按实际情况修正，但应保留变更原因和审计记录。

7. 账号与密码

账号来源

  • 开发账号通常由 GitHub Actions 生成，artifact 名称为 initial-person-accounts 或统一的 initial-accounts。
  • 管理员账号也可以写入统一的 initial-accounts.csv。
  • CSV 中 username 是登录账号，password 是初始密码，role 是权限角色。
  • 如果 password 为空，表示该账号已存在且本次没有重置密码；旧密码无法反查，只能使用原始 artifact 或重置。

分发建议

  • 不要把整份 CSV 发到群里。
  • 按人单独转发其账号和初始密码。
  • 提醒同学首次登录后立即点击 修改密码。
  • artifact 有过期时间，管理员应下载后妥善保存，避免过期后找不到初始密码。

密码安全

  • 网站不会明文存储密码；后端保存的是加盐哈希。
  • 用户忘记密码时无法反查，只能由管理员重置。
  • 管理员重置账号密码属于高风险操作，应确认身份后再执行。

8. PR 同步与实时刷新

PR 候选池

  • 系统每天北京时间 08:00 和 14:00 扫描 flashserve/flash-linear-attention-npu 仓库 PR。
  • 候选池包含已合入 PR 和未合入但仍打开的 PR；关闭但未合入的 PR 不纳入候选。
  • 页面选择 PR 时支持输入 PR 号快速匹配，避免下拉列表过长。
  • 已经关联的 PR 也可以重复关联到其他任务；一个任务也可以关联多个 PR。

数据实时刷新

  • 前端会周期性检查后端版本。
  • 如果没有本地未保存修改，后台有新版本时页面会自动刷新。
  • 如果本地有未保存修改，页面不会自动覆盖，会提示先刷新或手动合并。
  • 所有写操作会写入审计日志，可在“最近变更”查看最近 10 条。

9. 常见问题

为什么我登录后只看到一部分任务？

开发账号只展示自己负责的任务，以及当前时间窗口内与自己任务属于同一算子的关联任务。如果窗口内没有自己的任务，关联任务也不会额外展示。

为什么我不能拖动甘特条？

只有管理员可以调整排期。开发账号只能维护自己的 PR 链接和转测报告。

为什么保存时要求填写排期变更原因？

排期变化会影响项目承诺和人力安排，因此必须备注原因，便于后续审计和复盘。

为什么 PR 已填但风险没变低？

需要确认 PR 是否在候选池中、是否已合入、是否保存成功，以及 DDL 距离是否触发规则。没有转测报告时，即使 PR 已合入也不代表完成。

为什么任务显示 delay？

到达 DDL 当天 24:00 后，如果 PR 未合入或转测报告未填写，任务会进入 delay。甘特图会用白底红框展示延期阶段，直到交付件完备。

管理员账号能多人共用吗？

不建议。多人共用会导致审计日志无法区分真实操作者，也更容易出现并发修改冲突。建议每位管理员使用独立账号。
